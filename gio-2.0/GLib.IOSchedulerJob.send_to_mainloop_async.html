send_to_mainloop_async

Description:

[ Deprecated ]
public void send_to_mainloop_async (owned SourceFunc func)

Warning: send_to_mainloop_async is deprecated.

Used from an I/O job to send a callback to be run asynchronously in the thread that the job was started from.

Note:

Use g_main_context_invoke.

The callback will be run when the main loop is available, but at that time the I/O job might have finished. The return value from the callback is ignored.

Note that if you are passing the user_data from g_io_scheduler_push_job on to this function you have to ensure that it is not freed before func is called, either by passing null as notify to g_io_scheduler_push_job or by using refcounting for user_data.

Parameters:

this

a IOSchedulerJob
func

a SourceFunc callback that will be called in the original thread

notify

a DestroyNotify for user_data, or null
user_data

data to pass to func
